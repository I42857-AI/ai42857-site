🏗 二层架构（与 WB 的差异）

Marvis 沙箱将数据源和实体层合一（均由 ~/.agents/ hb 仓库承担），虫洞层做最终注入。相比 WorkBuddy 的三层架构（/workspace → /root → 虫洞），少了一层物理分发但有虫洞层作为最终出口。

关键设计思路：为什么 Marvis 用二层架构

WorkBuddy 的三层架构（/workspace → /root → /trae-cn）是因为飞书沙箱有独立的实体层空间分配需要。
Marvis 的 hb 仓库本身就是 Git 远程仓库的本地克隆，天然承担了"数据源"和"实体层"双重角色。
虫洞层 ~/.trae-cn/skills/ 通过 Symlink 指向 ~/.agents/skills/，系统按规范自动注入上下文。
多 Agent 应用目录（.qclaw / .marvis）在 hb 仓库根目录下，上传时优先推送，虫洞化注入真实读取目录。
