Architecture

PC 域化架构

4.0 核心虫洞：C:\Users\PC → Z:\PC\ → Z:\XiangMu-KongJian\，独立用户域，等效于 Z 盘化。

三层映射架构

PC 域 = 独立用户域，等效于 Z 盘化（盘符固定 + 用户名固定）。所有应用通过 PC 域自动获得正确的环境变量和路径，C 盘只留一个 Symlink：C:\Users\PC → Z:\PC\，其余全部自动生效。

层级 移动盘位置 Windows 映射 职责
用户配置层 Z:\PC\ C:\Users\PC\ 用户域数据（dotfile + 系统目录）
漫游数据层 credentials\{账号}\Roaming-* C:\Users\{用户}\AppData\Roaming 应用漫游数据
应用管理层 Z:\XiangMu-KongJian\ 盘符\多层级散落\{APPNAME} 应用关联文件
⚠️ Z:\PC\ 与 Z:\XiangMu-KongJian\ 是同级目录，不是父子关系！
❌ Z:\XiangMu-KongJian\PC\.rizhi\（错误）
✅ Z:\PC\.rizhi\（正确）

动态检测（五变量，禁止硬编码）

所有脚本必须通过动态检测获取环境变量，禁止硬编码盘符、用户名等值。Z盘化后盘符稳定命中 Z:。

变量 获取方式
盘符 Z→D 倒序遍历找 XiangMu-KongJian
用户名 $env:USERNAME
机器名 $env:COMPUTERNAME
Node.js Get-ChildItem "$ProjectRoot\node\node-v*" -Directory | Select-Object -First 1
PC 域 C:\Users\PC → Z:\PC\
ProcessStartInfo 注入（启动应用必须）：
USERPROFILE = C:\Users\PC
APPDATA = C:\Users\PC\AppData\Roaming
LOCALAPPDATA = C:\Users\PC\AppData\Local

虫洞化汇聚

总控通过 Symlink 汇聚各项目数据。一处更新全局生效——项目实体随项目走，总控只持有 Symlink 指向。

总控目录 功能 说明
Z:\PC\.rizhi\ 日志总控 Symlink 指向项目目录下的实体，不持有数据
Z:\PC\.jiaoben\ 脚本快速调用 Symlink 指向项目 JiaoBen 目录
Z:\PC\.agents\ 技能体系 30 个字母化技能目录，唯一真实数据源
Z:\PC\ZhiShi-TuPu\ 知识图谱总控 一站式查阅，断链无需维护
5.0 虫洞命名逻辑：目录 .全小写拼音，文件 {AppName}-PascalCase拼音
删除项目 → 实体自然消失 → 总控 Symlink 断链 → 苍穹自动标记"已卸载" → 零维护

Coding Safety

BAN/HAB 速查

BAN = 不可逆损坏，零容忍；HAB = Agent 坏习惯，3次+重复需纠正。

🚫 BAN 禁止规则

▶ BAN-001 禁止 PowerShell Set-Content 写中文

❌ Set-Content -Path "config.json" -Value $jsonContent # 原始字节永久丢失 ✅ Node.js: fs.writeFileSync(path, data, 'utf8') # 必须用 Node.js ✅ 回退: [System.IO.File]::WriteAllText($path, $content, [System.Text.UTF8Encoding]::new($true))

▶ BAN-002 禁止 Set-Content 修改脚本文件

❌ Set-Content -Path "script.ps1" -Value $newContent ✅ 必须用 SearchReplace 工具（Edit tool）修改脚本文件

▶ BAN-003 禁止 $env: + Start-Process 注入环境变量

❌ $env:USERPROFILE = "C:\Users\PC"; Start-Process $exe # 环境变量泄漏到父进程 ✅ $psi = [System.Diagnostics.ProcessStartInfo]::new($exe) $psi.EnvironmentVariables["USERPROFILE"] = "C:\Users\PC" [System.Diagnostics.Process]::Start($psi)

▶ BAN-004 禁止 Junction (mklink /D)

❌ cmd /c mklink /D "C:\link" "Z:\target" # Junction 不支持相对路径、跨盘符行为异常 ✅ New-Item -ItemType SymbolicLink -Path "C:\link" -Target "Z:\target" ✅ cmd /c mklink "C:\link" "Z:\target" # 无 /D = SymbolicLink

▶ BAN-005 禁止双引号中 ${Var}: 不加 ${}

❌ "$AppName: OK" # 解析阶段语法错误，脚本闪退 ✅ "${AppName}: OK" # 变量后紧跟冒号等非变量名字符必须用 ${}

⚠️ HAB 坏习惯

▶ HAB-001 默认用 PowerShell 处理中文文本

❌ Get-Content / Set-Content 处理中文 JSON/文本 ✅ 一律用 Node.js（fs.writeFileSync + utf8）

▶ HAB-002 默认用 mklink /D 创建 Junction

❌ mklink /D → Junction ✅ 用 New-SafeSymLink 或 mklink（无 /D）→ SymbolicLink

▶ HAB-003 默认 dot-source 在函数调用之后

❌ Invoke-MyFunc; . .\lib.ps1 # 函数未定义就调用 ✅ . .\lib.ps1; Invoke-MyFunc # 先 dot-source 再调用

▶ HAB-004 默认用 robocopy /MOVE 一步到位

❌ robocopy /MOVE（复制+删除一步完成，失败时数据丢失） ✅ 先 robocopy 复制 → 验证文件完整性 → 再删除源目录
