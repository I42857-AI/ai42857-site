P0 任务衔接（回访-日志联动，强制执行）
================================================================================

  ⚠️ 本条为 P0 最高优先级，所有 Agent 在完成实质性任务后必须执行，不可跳过。
  回访和日志是你和我（用户）和其他 Agent 沟通无障碍/协作的前提。

  核心原则：每次完成一个实质性任务后，必须主动使用 AskUserQuestion 工具弹出回访问卷，
  让用户选择下一步方向，避免重新排队等待新对话。

  ⚠️ 禁止依赖「其他(Other)」字段传递自定义文本：
  AskUserQuestion 的「其他」选项无法可靠传递用户的自定义输入。
  调用方只能收到选项标签，无法获取用户实际填写的内容。
  正确做法：自定义需求通过后续聊天输入获取，不使用「其他中描述」模式。

  无需回访模式：
  默认必须弹窗。仅当用户明确说「无需回访」时跳过弹窗，自然衔接。
  用户说「恢复回访」或切换模型后，恢复弹窗机制。

  必弹场景：任务完毕 / 验证通过 / 用户确认后 / 用户说"好了""完成" / 遇到歧义需要确认 / 发现更优方案需征求用户意见
  不弹场景：多步骤流程中间 / 用户说"不用了""结束" / 用户已说「无需回访」

  深度回访问卷（2问，上下文感知创造力触发，详见 CanKao/00-shenDu-huiFang.md）：
    第一问：创造力触发（3选项，从当前上下文动态生成）
      从7种触发类型中选3种：推到边界/找矛盾/换隐喻/回原点/反直觉/往下挖/换立场
      选项文案 = 触发词（2-4字）+ 具体提示（一句话，引用当前任务内容）
      没有降级——简单问答更要逼出创造力选项
    第二问：自由方向（3选项）
      ├─ 补充/深化当前任务
      ├─ 开始新任务
      └─ 我有自己的方向（用户自由输入）

  问卷设计原则：
  - 选项 = 提示词：用户点一下，Agent 直接进入对应模式
  - 第一问必须引用当前任务的具体内容，不是泛泛而谈
  - 最多 2 个问题：避免多轮交互导致排队
  - 第二问用户可跳过（选了第一问就直接执行）
  - 用户直接在聊天中给出下一步指令 → 优先执行指令，完成后再次弹问卷

  日志联动机制（回访-日志-提炼联动，强制执行）：
    任务完成 → 是否实质性工作？
      ├─ 否（简单问答：信息查询、单行修改、纯对话）→ 直接回访
      └─ 是（优化/升级/阶段性成果：规范更新、脚本修复、架构调整、数据迁移）
           → 写入日志 → 更新提炼日志 → 回访
                ├─ 有 AskUserQuestion → 弹窗回访
                └─ 无 AskUserQuestion → 文字追问（自动降级）

  日志写入规则：
    格式：YAML 元数据（id/author/timestamp/status/scope）+ 正文（问题→方案→变更→验证）
    规则：追加不覆盖 / 禁止硬编码盘符 / 中文优先 / 页眉自解释 / 编入者为变量
    时机：回访前判断，实质性工作必须写入，简单问答跳过

    ━━━ 日志分层架构（虫洞化汇聚）━━━

P0 编码安全（核心规则，详解见 CanKao/04-coding-safety.md）
================================================================================

  中文编码：Linux 默认 UTF-8，无乱码风险。.sh 文件名+内容保持拼音字母化。

  双引号变量展开铁律（Bash 版）：
    变量后紧跟冒号等非变量名字符 → 必须用 ${} 包裹
    ❌ `echo "$AppName: OK"` → 可能解析错误
    ✅ `echo "${AppName}: OK"`

  ━━━ BAN（禁止，不可逆损坏，零容忍）━━━
    BAN-001: 禁止 rm -rf 非空目录（除非用户明确确认）→ 必须 BAK 化
    BAN-002: 禁止覆盖写入脚本文件 → 必须用 Edit 工具
    BAN-003: 禁止硬编码路径 → 必须用变量或动态检测
    BAN-004: 禁止 Junction → 唯一链接类型 = SymbolicLink (ln -s)
    BAN-005: 禁止 find \| python 的 lstrip('./') → 会剥离隐藏目录前导点
    BAN-006: 禁止在已有全量 checkout 的仓库上执行 git sparse-checkout init → 会重置工作目录，其他文件全部消失。恢复：git sparse-checkout disable && git checkout HEAD -- .

  ━━━ HAB（坏习惯，Agent 默认行为，3次+重复需纠正）━━━
    HAB-001: 默认用 find \| python 生成目录树 → 纠正：用 os.listdir() + os.path.islink()
    HAB-002: 默认复制文件到实体层 → 纠正：用 Symlink 指向数据源（零重复）
    HAB-003: 默认硬编码 UTC 时间 → 纠正：用 TZ=Asia/Shanghai date 获取北京时间
    HAB-004: 默认用 cp -r 同步技能 → 纠正：用 ln -s 虫洞化链接
