# 30秒AI短视频 · 分镜操作系统

> "一道菜的一生只有30秒。每一帧都必须让人流口水。"
> —— 源自 MrBeast「零无聊时刻」的极限压缩：观众手指永远悬在「下一个视频」上，你只有0.5秒让人停下来。

## 回答工作流（Agentic Protocol）

### Step 1: 问题分类 + 前期调研

收到问题后，先判断类型：

| 类型 | 特征 | 行动 |
|------|------|------|
| **需要分镜** | 用户提到某道菜需要做30秒宣传视频 | → 进入 Step 1.5 前期调研，再进入 Step 2 |
| **需要优化** | 用户已有分镜但需要调整/优化 | → 对照检查清单逐条审查，给出具体修改建议 |
| **需要提示词** | 用户需要画面提示词 | → 按提示词工程指南生成，严格遵循专用后缀 |
| **一般咨询** | 询问方法论/技法/色调等 | → 直接用本技能框架回答 |

### Step 2: 封面帧先行设计（缩略图先行产线）

> **核心原则**（源自 MrBeast 缩略图先行理论）：先做封面帧，再写分镜。
> 封面帧决定 CTR（观众是否停下来），分镜决定 AVD（观众是否看完）。
> 大多数创作者先做视频再想封面——反过来。封面帧是整个视频的"概念即画面"。

**美食缩略图五原则**（从 MrBeast 五原则转化，详见 `CanKao/research/03-thumbnail-first-theory.md`）：

| # | 原则 | 美食应用 | 封面帧体现 |
|---|------|---------|-----------|
| 1 | 极简构图（Zoom Out Test） | 缩小到手机通知栏仍能识别 | 极浅景深+背景全暗+单一焦点 |
| 2 | **菜品替代面部** | 脆皮截面/流心涌出 = 菜品的"表情" | 质感极致呈现替代表情夸张 |
| 3 | 大物体法则 | 菜品占画面40-50% | 截面/特写占主体，餐具只作参照 |
| 4 | 色调与菜品绑定 | 烧腊暖金/蒸品冷白/炒类橙红 | 绝不用灰暗色调 |
| 5 | **质感引导** | 油光/汁水/蒸汽方向 = 引导线 | 替代手臂/箭头/目光引导 |

**封面帧 = 抖音信息流中视频静止时显示的那一帧。观众在 0.5 秒内决定是否停留。**

#### 2.1 确定最诱人角度

| 菜品类型 | 最诱人角度 | 封面帧画面 |
|---------|-----------|-----------|
| 烧腊类 | 皮肉切面 | 刀切烧鹅，皮脆肉嫩的横截面，油光反光 |
| 蒸品类 | 蒸汽揭盖 | 揭盖瞬间蒸汽升腾，鱼肉/鸡身完整呈现 |
| 炒类 | 镬气翻腾 | 食材在锅中翻飞，火焰包裹 |
| 煲仔类 | 酱汁淋落 | 酱汁从勺中缓缓淋下，表面微微起伏 |
| 点心类 | 破坏瞬间 | 筷子夹起虾饺，皮破露出虾肉/流沙包撕开 |

#### 2.2 竖屏构图法则（9:16）

```
┌──────────────┐
│   留白区      │  ← 顶部 1/6：品牌/标题区（可选）
│              │
│  ┌────────┐  │
│  │ 视觉焦点 │  │  ← 中部 2/3：菜品主体
│  │  菜品   │  │     构图重心偏上 1/3
│  └────────┘  │
│              │
│   质感区     │  ← 底部 1/6：汁水/碎屑/蒸汽
└──────────────┘
```

**竖屏三原则**：
1. **单焦点**：画面中只有 1 个视觉焦点（菜品本身），不要出现人手/餐具/环境
2. **上重下轻**：构图重心在上 1/3，底部留出汁水/蒸汽的延伸空间
3. **极浅景深**：背景全虚，只有菜品清晰——缩小到手机屏幕时仍然醒目

#### 2.3 封面帧色调匹配

直接使用粤菜色调体系（见第三章），封面帧色调 = 整片色调。

#### 2.4 封面帧 Seedance 提示词模板

```
[菜品名称]，[最诱人角度描述]，[色调名称]色调，极浅景深，背景全暗，
[光源方向]侧光，[质感关键词]，蒸汽/油光粒子，9:16竖屏构图，
极致特写，无人物出镜，无文字叠加
```

**示例**（烧鹅封面帧）：
```
烧鹅，刀切横截面皮脆肉嫩，烧腊暖金色调，极浅景深，背景全暗，
左上45度侧光，酥脆表皮金黄油亮肉质粉嫩多汁，油光粒子，9:16竖屏构图，
极致特写，无人物出镜，无文字叠加
```

#### 2.5 封面帧→钩子镜头对齐

**封面帧必须是钩子（0-5秒）的某一帧。** 观众点击后看到的第一个画面 = 封面帧的动态版本。
这保证了"所见即所得"，避免封面帧与实际内容不符导致的跳出。

### Step 3: 分镜生成流程

1. **确认菜品类型**：烧腊/蒸品/炒类/煲仔/点心，选择对应色调和分镜要点（参考 `CanKao/cuisine-types.md`）；同时确认分镜类型（食欲炸弹/工艺解密/情感共鸣/剪辑炫技/感官沉浸/反差爆破），选择对应叙事骨架和情绪曲线（参考 `CanKao/storyboard-types.md`）
2. **按五幕结构分配时长**：钩子(5s) → 张力(5s) → 爆发(8s) → 收束(8s) → 落版(4s)
3. **逐镜填写12字段**：严格按字段规范，无内容填「无」
4. **生成画面提示词**：每镜一段画面内容提示词（全局参数由RunningHUB预设，仅写画面内容）
5. **自检**：对照检查清单逐条验证

### Step 3: 输出格式

分镜输出必须以 **HTML 网页格式** 交付，遵循 `html-report-design-system` 技能规范。

**HTML 页面结构**：

1. **固定导航栏**：可跳转至各幕（钩子/张力/爆发/收束/落版）和提示词区域
2. **Hero 区域**：菜品名称 + 五幕结构概览
3. **分镜表格**（使用 `.wide-table-wrap` 全宽表格，12列，每列对应一个字段）
4. **提示词区域**：
   - **一键复制全部分镜提示词**按钮：将所有镜号的画面提示词合并为一段文本，点击即复制
   - **按镜号单独复制**：每个镜号的提示词独立展示，配有单独的「复制」按钮
5. **手机端适配**：宽表格自动转为卡片式布局

**提示词复制交互**：
- 点击「复制全部提示词」→ 将全部镜号提示词合并复制到剪贴板，按钮变为「✓ 已复制」
- 点击单个镜号的「复制」→ 仅复制该镜号的画面提示词，按钮变为「✓ 已复制」
- 2秒后按钮自动恢复

### 缩略图描述规则

每个镜头必须填写**缩略图描述**——该镜头最精华的一帧静态绝美诱人的美食显性状态。

> ⚠️ **缩略图描述 ≠ 画面描述的重复**。画面描述描述的是动态过程（"蜜汁刷过烧鹅表皮，金黄光泽沿弧面流动"），
> 缩略图描述捕捉的是过程中最诱人的一帧定格（"蜜汁覆盖的烧鹅表皮，金黄光泽凝固在弧面最高点"）。
> 好的缩略图描述 = 按下暂停键时，那一帧画面必须让人咽口水。

**缩略图描述三原则**：
1. **静态定格**：描述的是一帧，不是过程。用"凝固""定格""瞬间"等词暗示静态
2. **绝美诱人**：只保留最精华的视觉元素，删除一切过渡/过程/动作描述
3. **显性状态**：描述美食的显性特征（色泽/质感/形态），不描述隐性特征（味道/温度/声音）

**缩略图吸引力测试**（源自 MrBeast 缩略图先行理论 + 美食缩略图五原则）：
- 单焦点：画面中只有 1 个视觉焦点
- 自解释：不需要上下文就能感受到食欲
- 缩小测试：缩小到手机屏幕大小仍然诱人
- 菜品替代面部：缩略图中的菜品状态 = 菜品的"表情"
- 质感引导：油光/汁水/蒸汽方向形成视觉引导线

**对比示例**：
| | 画面描述（动态过程） | 缩略图描述（静态定格） |
|---|---|---|
| A1 | 刚出炉的烧鹅胸截面，皮肉分离处油光闪烁，热气微微升腾 | 金红烧鹅截面凝固特写，皮脆肉嫩层次分明，油光定格 |
| A2 | 蜜汁刷过烧鹅表皮，金黄光泽沿弧面流动 | 蜜汁覆盖的烧鹅表皮，金黄光泽凝固在弧面最高点 |
| B3 | 刀切入烧鹅脆皮，皮肉分离处油光 | 刀口处脆皮裂开定格，皮肉分离截面油光闪烁 |
| B5 | 刀切入脆皮，"咔嚓"一声皮裂汁出 | 刀口处汁水渗出定格，脆皮裂纹与汁水反光 |

### 字段填充规则

| 字段 | 规则 |
|------|------|
| 镜号 | 格式 AN/BN（A=Clip A诱惑篇，B=Clip B呈现篇），如 A1、A2、B1、B2 |
| 画面描述 | 必须以【技法标签】开头，如【极致特写+慢推镜】【跳切省略法】【慢动作粒子】 |
| 角色 | 格式 `角色名:描述\|动作`，无人时填「无」。如"主厨:手部粗犷有力\|翻锅抖腕" |
| 参考景别 | 微距特写/大特写/特写/中特写/中景/全景，可写过渡如「特写→大特写」 |
| 场景标签 | 3-5个标签，标签间用顿号分隔 |
| 分镜提示词 | 用顿号分隔2-4个技法标签，不可为空 |
| 运动提示词 | 必须包含具体的镜头运动指令（方向、速度、时长），不可为空 |
| 缩略图描述 | 必须通过缩略图吸引力测试（单焦点+自解释+缩小测试） |

## 二、核心设计哲学

### 源头：MrBeast 六个模型的迁移适配

| MrBeast 原模型 | 粤菜30秒映射 | 应用方式 |

### 分形结构：15秒×2 产线（Seedance 适配）

> **Seedance 单次生成上限 15 秒。** 30 秒成片 = 2 段 15 秒片段拼接。
> 每段 15 秒必须**独立完整**——单独看也能吸引人看完。
> 这就是分形：小整体组成大整体，每个层级都有完整的吸引力。

```
30s 完整体（大整体）
├── Clip A (0-15s) "诱惑篇" ── 独立可看，结尾有 mini-climax
│   ├── 钩子 (0-5s)   成品砸脸，极美特写
│   ├── 张力 (5-10s)  食材质感，建立期待
│   └── 爆发前半 (10-15s)  烹饪启动，火力初现
│       └── 15s 处：mini-climax（汁水四溅/油花爆裂/蒸汽升腾）
│
└── Clip B (15-30s) "呈现篇" ── 独立可看，开头有视觉钩子
    ├── 爆发后半 (15-18s)  烹饪高潮，视觉能量峰值
    ├── 收束 (18-26s)  精妙装盘，酱汁淋落
    └── 落版 (26-30s)  终极呈现，破坏性动作
        └── 30s 处：终极定格（最诱人瞬间）
```

**分形三原则**：

1. **Clip A 独立性**：结尾必须有 mini-climax（不是悬念！是满足感）
   - 单独看 = "这道菜太诱人了"（满足）
   - 接着看 = "后面还有更精彩的"（期待）
   - ❌ 错误：15s 处留悬念 → 单独看 Clip A 的观众觉得没完

2. **Clip B 独立性**：开头必须有视觉钩子（不依赖 Clip A 上下文）
   - 单独看 = "这道菜做出来了"（满足）
   - 连着看 = "从诱惑到呈现的完整旅程"（更满足）
   - ❌ 错误：Clip B 开头是过渡画面 → 单独看时观众滑走

3. **每帧缩略图测试**：每个镜头的关键帧必须通过缩略图吸引力原则
   - 单焦点：画面中只有 1 个视觉焦点
   - 自解释：不需要上下文就能感受到食欲
   - 极致质感：缩小到手机屏幕仍然诱人

### 无字幕的纯视觉叙事策略

- [ ] 随机截取第一镜的某一帧单独作为封面，会让人想吃吗？
- [ ] 如果不能 → 钩子画面需要更强的质感或更诱人的角度。

- [ ] 30秒内的每一帧单独拉出来看——如果有一帧让人觉得「普通」→ 删掉或替换。

- [ ] 画一条「食欲刺激强度曲线」：钩子(高) → 张力(略低但上升) → 爆发(峰值) → 收束(缓降但精致) → 落版(次高峰)

- [ ] 菜品本身也许只是一道普通粤菜，但30秒的呈现必须让人感觉这是「人生必吃」。

## 七、抖音算法优化

### 3 秒完播率（抖音第一指标）

抖音算法最看重 **3 秒完播率**（观众在 3 秒内不滑走的比例）。这比 YouTube 的 CTR 更重要——因为抖音是自动播放，没有"点击"这个动作，只有"停留"。

**3 秒内必须做到**：
1. **第 0-1 秒**：封面帧动态化——观众看到的就是封面帧开始动（蒸汽升腾/油光闪烁/汁水流动）
2. **第 1-2 秒**：微动作揭示——筷子触碰/刀刃切入/酱汁滴落，一个极小的动作暗示"即将发生什么"
3. **第 2-3 秒**：第一个破坏性动作——皮被夹起/盖被揭开/食材入锅，食欲瞬间引爆

### 完播率优化（30 秒全片）

| 时段 | 目标完播率 | 策略 |
|------|-----------|------|
| 0-3s | >85% | 封面帧动态化 + 微动作揭示 |
| 3-10s | >70% | 食材质感冲击，建立"这道菜值得看完"的信念 |
| 10-18s | >55% | 烹饪高潮，视觉能量峰值 |
| 18-26s | >45% | 精妙收尾，节奏放缓但质感升级 |
| 26-30s | >35% | 破坏性动作 + 终极呈现，驱动互动 |

## 八、提示词工程指南

> **注意**：全局前缀（模型、风格、画质等参数）由 RunningHUB 平台预设，无需在提示词中重复填写。
> 以下指南仅针对每镜的画面内容提示词编写。

### 各幕专用后缀

| 幕 | 专用后缀 |
|----|---------|
| 钩子 | 极致特写，极浅景深，背景全暗，蒸汽/油光粒子，慢速推镜，无人物出镜，无文字叠加 |
| 张力 | 食材质感，自然清新，砧板/竹篮，水珠反光，慢速横移，无文字叠加 |
| 爆发 | 动态激烈，火光/蒸汽/油花，慢动作粒子，高频细节，无文字叠加 |
| 收束 | 仪式感，酱汁流动，极慢镜头，盘边洁净，暖光顶打，无文字叠加 |
| 落版 | 完整呈现，黄昏暖光，破坏性动作，内在质感，定格，无文字叠加 |

### 提示词编写原则

1. **具体胜过抽象**：不说「看起来很香」，说「蜜汁表面形成镜面反光，边缘微焦冒小泡」。
2. **光线就是食欲**：食物的诱惑力90%来自光线。每次提示必须明确光源方向和色温。
3. **质感优先于构图**：先描述表面质感（酥脆/滑嫩/起沙），再描述画面布局。
4. **运动状态不能模糊**：AI需要明确的运动指令——「极慢推镜」而非「镜头动一下」。

### ⚠️ 反面提醒：禁止"油光泛滥"

**问题**：大量分镜的质感描述集中在"油光""油滴""油珠"——所有菜品看起来都是热量炸弹，缺乏质感多样性。

**根因**：烧腊类（暖金色调）天然倾向油光描述，但其他菜品类型各有独特的质感语言。

**纠正**：每种菜品类型必须使用该类型专属的质感关键词，而非统一套用"油光"：

| 菜品类型 | ❌ 禁止滥用 | ✅ 应使用 |
|---------|-----------|---------|
| 烧腊类 | 油光闪烁×N | 酥脆/焦糖化/琥珀/蜜光/烟熏色 |
| 蒸品类 | 油光/油亮 | 嫩滑/半透明/晶莹/水润/葱丝翠绿 |
| 炒类 | 油花四溅×N | 镬气/焦边/干身/锅气/弹牙 |
| 煲仔类 | 油滋滋 | 滋滋冒泡/焦脆锅巴/蒸汽/啫啫声 |
| 点心类 | 油光/油亮 | 半透明/粉嫩/弹滑/流心/酥层 |

**自检规则**：生成分镜后，搜索"油"字出现次数——超过 3 次/9 镜 = 触发此警告，必须替换为菜品类型专属质感词。

### ⚠️ 反面提醒：禁止"冒泡""焦化"等 AI 误导词

**问题**：提示词中写"冒泡""焦化"等描述，AI 视频生成会**字面放大**这些特征，导致：
- "冒泡" → 密集恐惧感，满屏气泡翻滚
- "焦化" → 菜品整体发黑发糊，看起来烧焦了
- "炸裂" → 油花飞溅过度，画面混乱

**根因**：AI 视频生成 ≠ 实拍。实拍中"冒泡"是自然的微观现象，AI 会将其放大为宏观画面。提示词必须描述**期望的最终画面**，而非烹饪过程的物理现象。

**纠正**：

| ❌ 禁止使用 | 会导致 | ✅ 替换为 |
|-----------|--------|---------|
| 冒泡 | 密集气泡翻滚，密集恐惧 | 微微起伏/表面轻颤/蜜光流动 |
| 焦化 | 菜品发黑发糊 | 金黄渐变/琥珀色边缘/微焦边缘 |
| 炸裂 | 油花飞溅过度 | 轻微溅跃/细碎油星/闪烁微光 |
| 碳化 | 黑色覆盖 | 深褐焦香/焦糖色泽 |
| 沸腾 | 液体剧烈翻滚 | 微沸/轻柔翻涌/热气升腾 |

**原则**：提示词描述的是**画面应该长什么样**，不是**烹饪过程中发生了什么物理反应**。AI 不理解"焦化是好的"，它只会把"焦"画得更焦。

## 九、本技能运用的核心技法（师承 MrBeast）

| 技法 | MrBeast 源头 | 在粤菜视频中的运用 |
|------|-------------|------------------|
| 成品砸脸法则 | 概念即画面 | 第0-5秒直接展示成品最美瞬间，不做任何铺垫 |
| 跳切省略法 | 零无聊时刻 | 砍掉所有中间过程，只保留动作起点和终点 |
| 慢动作粒子法则 | 视觉预告 | 火焰/油花/蒸汽用120-240fps慢镜强调 |
| 阶梯递进曲线 | Stair-Stepping | 全片食欲刺激度逐镜爬升，不允许平台期 |
| 静默高潮蓄力 | 节奏控制 | 落版前1-1.5秒完全静默，然后一击引爆 |
| 破坏性动作揭示 | 传达>内容 | 筷子夹起/刀切/戳破等破坏动作展示内在质感 |
| 质感优先原则 | 极端执行 | 每一镜都追求食材表面的极致质感表现 |
| 声画同步叙事 | CTR×AVD | 声音能量曲线与画面刺激度严格同步 |

## 十、生产工作流（分镜→成片）

### 全链路流程

```
封面帧设计 → 分镜生成(分形15s×2) → Seedance提示词(2组) → AI视频生成(2段) → 剪辑拼接 → 抖音发布
   Step 2        Step 3              Step 3.4          RunningHUB     剪映/PR    抖音App
```

### 各环节操作指南

| 环节 | 工具 | 输入 | 输出 | 注意事项 |
|------|------|------|------|---------|
| 封面帧设计 | 本技能 Step 2 | 菜品名称 | 封面帧提示词 | 先做封面帧，再写分镜 |
| 分镜生成 | 本技能 Step 3 | 菜品+分镜类型 | 12字段分镜表(分形2组) | 钩子对齐封面帧，15s处有mini-climax |
| Seedance 提示词 | 本技能 Step 3.4 | 分镜表 | 2组提示词(Clip A/B) | 每组≤15s，全局参数由平台预设 |
| AI 视频生成 | RunningHUB 平台 | 2组提示词 | 2段视频片段 | 9:16 竖屏，每段≤15s |
| 剪辑拼接 | 剪映/PR | 2段视频+音乐 | 30s 成片 | Clip A结尾+Clip B开头自然过渡 |
| 抖音发布 | 抖音 App | 成片+文案+标签 | 发布 | 封面帧选钩子第一帧 |

### 分形剪辑要点

**Clip A → Clip B 过渡**：
- 画面过渡：Clip A 的 mini-climax 画面 → Clip B 的视觉钩子（同色调/同菜品，自然衔接）
- 音频过渡：Clip A 结尾音效渐弱 → Clip B 开头音效渐入（不要硬切）
- 节奏过渡：Clip A 结尾节奏加快 → Clip B 开头节奏稍缓再加速（呼吸感）

**独立发布场景**：
- Clip A 可单独作为"预告片"发布（抖音故事/朋友圈）
- Clip B 可单独作为"成品展示"发布（不同账号/不同时段）
- 两段都通过缩略图测试 → 任何一段被推荐都能独立吸引观众

### 批量生产模式

多道菜品批量产出时，按以下顺序提高效率：

1. **批量封面帧**：一次性设计所有菜品的封面帧提示词
2. **批量分镜**：同类型菜品共用叙事骨架，仅替换菜品特有画面
3. **批量提示词**：同色调菜品共用后缀，仅替换画面内容
4. **批量生成**：Seedance 批量提交，等待生成
5. **批量剪辑**：同模板剪辑，替换视频片段和音乐
