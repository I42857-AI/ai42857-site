# 文生图（WenShengTu）

基于 TRAE 内置 text_to_image API + 豆包 Seedream 5.0 模型的文生图技能。

## API 信息

- **URL**：`https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt={URL编码的提示词}&image_size={image_size}`
- **模型**：豆包 Seedream 5.0（大概率 Doubao-Seedream-5.0-Lite）
- **CDN**：aka.doubaocdn.com → 字节跳动 imagex，短链接快速过期
- **认证**：需 TRAE 内部认证，外部脚本无法直接调用

### image_size 分辨率对照表

| image_size | 实际分辨率 | 宽高比 | 适用场景 |
|------------|-----------|--------|----------|
| square_hd | 1024x1024 | 1:1 | 商标/图标/单品特写 |
| square | 1024x1024 | 1:1 | 同上 |
| landscape_16_9 | 1368x768 | 16:9 | 菜品/场景/海报（默认横版） |
| landscape_4_3 | 1216x912 | 4:3 | 产品渲染/建筑空间 |
| portrait_4_3 | 912x1216 | 3:4 | 人物半身像 |
| portrait_16_9 | 768x1368 | 9:16 | 人物/非菜品默认竖版 |

⚠️ 非官方参数（4k/ultra_hd/1920x1080等）全部回退为 1024x1024

### 默认 image_size 规则

| 类型 | 默认 image_size | 说明 |
|------|----------------|------|
| 菜品 | landscape_16_9 | 横版展示菜品 |
| 人物 | portrait_16_9 | 竖版展示人物 |
| 商标/图标 | square_hd | 方形适合商标 |
| 海报 | landscape_16_9 或 portrait_16_9 | 视内容而定 |
| 产品渲染 | landscape_4_3 | 4:3适合产品展示 |
| 场景 | landscape_16_9 | 横版展示场景 |

## 提示词铁律（P0 强制）

1. **中文原文不翻译**：提示词使用中文，一字不改直接使用，不翻译不精简
2. **必须指定模型**：尾部追加"使用豆包Seedream 5.0模型生成"，避免降级
3. **分图生成**：不做合图构图指令，豆包无法执行多格布局
4. **无文字例外**：菜品/场景加"无文字"，海报/商标加"无文字（""引号内文字除外）"
5. **超详细 > 精简**：核心特征描述200字 > 3-5关键词，豆包会偷懒

### 菜品模板
```
{菜名}提示词：{景别}特征
{特征描述}；状态提醒：{熟食状态而非生食状态}；生成{菜名}在物理世界的正视图，全景图，超写实，无裁切，纯白色背景。无人物，无文字，landscape_16_9，4K分辨率。使用豆包Seedream 5.0模型生成
```

### 人物模板（五段结构）
```
[身份锚定] + [超详细核心特征] + [服饰字段] + [变体动作/表情] + [品质参数]
```
- 身份锚定：名人名字锁定面部骨骼比例
- 超详细核心特征：脸型→额头→颧骨→下颌→眉→眼→鼻→唇→下巴→肤色→肤质→发型（200字）
- 服饰字段：根据景别动态调整描述范围（近景→领口/面料纹理；中景→上半身款式/面料；远景→全身搭配/鞋子）
- 变体：每张图不同的动作/表情/姿态
- 品质参数：相机+镜头+光圈+布光+胶片模拟+模型+分辨率

### 人物三策略组合（降低AI感）
1. **皮肤微观描述**：毛孔/绒毛/油光/血色/痣/胡茬
2. **参考摄影师风格**：风格锚定整体调性
3. **摄影参数注入**：相机型号+镜头+光圈+布光方案

### 场景模板
```
{场景名}，{构图设计：前景/中景/远景层次}，{色彩搭配：色值+渐变描述}，{光影氛围：光柱/逆光/色温}，{摄影师风格参考}，超写实，无文字，landscape_16_9，4K分辨率。使用豆包Seedream 5.0模型生成
```
⚠️ 场景核心杠杆 = 构图+色彩+光影，微观细节描述效果有限

### 商标模板
```
{形状}商标图标设计，{风格}风格，纯白色背景，{主体描述}，{字体描述：书体+笔画+颜色+排列}，{印章描述：形状+内容+边框+质感}；配色为{色1}、{色2}、{色3}，{风格}风格，矢量风格，商标设计，square_hd，4K分辨率。使用豆包Seedream 5.0模型生成
```

### 海报模板
```
{海报主题}海报设计，{构图描述}，{主视觉描述}，{文字排版描述}，{配色方案}，{风格参考}，无文字（""引号内文字除外），landscape_16_9，4K分辨率。使用豆包Seedream 5.0模型生成
```

## 通用后缀

非菜品默认后缀：
```
无文字（""引号内文字除外），{image_size}，4K分辨率。使用豆包Seedream 5.0模型生成
```

菜品默认后缀：
```
无人物，无文字，{image_size}，4K分辨率。使用豆包Seedream 5.0模型生成
```

## 工作流

### 生成图片
1. 根据类型选择提示词模板
2. 填写变量字段（中文原文，不翻译不精简）
3. 追加通用后缀（含 image_size + 模型指定）
4. URL编码提示词，拼接到 API URL
5. 在 HTML 中用 `<img>` 标签引用

### 本地预览与保存
1. 创建 HTML 页面，`<img src="text_to_image URL">` 引用图片
2. 用 Node.js 起本地 HTTP 服务托管 HTML（参见 JiaoBen/yulan-fuwu.js）
3. 在 TRAE 预览窗口或外部浏览器打开
4. 图片正常渲染后，右键 → 图片另存为 → 保存到本地

⚠️ CDN 短链接过期很快，图片需即时保存，不可持久引用

## 能力边界速查

**强项（★★★★）**：产品渲染、图标/UI、海报设计、艺术风格、材质表现、人物（三策略组合+超详细特征）
**弱项（★★~★★★）**：极简建筑、光学效果、中文字体、菜品熟食状态、丹宁面料、纪实场景

详细评分和面料排序参见 CanKao/nengli-bianjie.md

## 提示词五色标注

| 颜色 | 字段 | 含义 |
|------|------|------|
| 红色 | 身份锚定 | 名人名字或角色描述，锁定骨骼比例 |
| 紫色 | 核心特征 | 完整面部特征清单（200字） |
| 蓝色 | 服饰字段 | 根据景别动态调整（近景→领口；中景→上半身；远景→全身） |
| 橙色 | 变体 | 动作/表情/姿态变化 |
| 绿色 | 品质参数 | 相机+镜头+布光+模型+分辨率 |

## 关键限制

- 豆包每次生成是独立上下文，负向约束（"不改变面部结构"）无效
- 单图多视图/多人物布局无法精准执行，需分图生成后拼接
- 三视图需求拆分为3张独立图
- "左大图+右四宫格"是豆包最能理解的构图模板
- 服饰描述范围需与景别匹配，避免矛盾（近景特写不应出现鞋子/下半身描述）
