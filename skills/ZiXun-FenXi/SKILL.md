# 专业研究报告技能

## 概述

本技能产出专业、咨询级的 Markdown 格式研究报告，覆盖**市场分析、消费者洞察、品牌策略、财务分析、行业研究、竞争情报、投资研究和宏观经济分析**等领域。分两个阶段运作：

1. **阶段一 — 分析框架生成**：给定研究主题，产出严谨的分析框架，包含章节骨架、各章数据需求、分析逻辑和可视化计划。
2. **阶段二 — 报告生成**：在其他技能完成数据采集后，将所有输入综合为最终精修报告。

输出遵循麦肯锡/BCG咨询风格标准。报告语言遵循 `output_locale` 设置（默认：`zh_CN` 中文）。

## 数据真实性协议

**严格遵循规则**：报告中呈现的所有数据和图表中可视化的数据**必须**直接来源于提供的**数据摘要**或**外部搜索发现**。
- **禁止虚构**：不得编造、估算或模拟数据。如果数据缺失，声明"数据不可用"而非捏造数字。
- **可溯源**：每个主要论断和图表必须可追溯到输入数据包。

## 核心能力

- 从零开始**设计分析框架**，仅需研究主题和范围
- 将原始数据转化为结构化、高深度研究报告
- 遵循**"视觉锚点 → 数据对比 → 综合分析"**流程处理每个子章节
- 产出遵循**"数据 → 用户心理 → 战略启示"**链的洞察
- 嵌入预生成的图表并构建对比表格
- 生成按 **GB/T 7714-2015** 标准格式的行内引用
- 以 `output_locale` 指定的语言输出报告，采用专业咨询语调
- 根据领域（营销、财务、行业等）调整分析深度和结构

## 何时使用此技能

**在以下情况下始终加载此技能：**

- 用户要求市场分析、消费者洞察报告、财务分析、行业研究或任何咨询级分析报告
- 用户提供研究主题，需要在数据采集前构建结构化分析框架
- 用户提供数据摘要、分析框架或图表文件，需要综合成报告
- 用户需要专业咨询风格的研究报告
- 任务涉及将研究发现转化为结构化的战略叙事

# 阶段一：分析框架生成

## 目的

给定**研究主题**（如"Z世代护肤市场分析"、"新能源汽车行业竞争格局"、"品牌X消费者画像"），产出完整的**分析框架**，作为下游数据采集和最终报告生成的蓝图。

## 阶段一输入

| 输入 | 说明 | 必需 |
|------|------|------|
| **研究主题** | 要分析的问题或主题 | 是 |
| **范围/约束** | 地理范围、时间区间、行业细分、目标受众等 | 可选 |
| **特定角度** | 用户希望探索的特定角度或假设 | 可选 |
| **领域** | 分析领域：市场、财务、行业、品牌、消费者、投资等 | 推断 |

## 阶段一工作流程

### 步骤 1.1：理解研究主题

- 解析研究主题，识别**核心实体**（市场、品牌、产品、行业、消费者群体、金融工具等）
- 识别**分析领域**（营销、财务、行业、竞争、消费者、投资、宏观等）
- 根据领域确定**自然分析维度**：

| 领域 | 典型维度 |
|------|----------|
| 市场分析 | 市场规模、增长趋势、市场细分、增长驱动力、竞争格局、消费者画像 |
| 品牌分析 | 品牌定位、市场份额、消费者认知、营销策略、竞品对比 |
| 消费者洞察 | 人口统计画像、购买行为、决策旅程、痛点、场景分析 |
| 财务分析 | 宏观环境、行业趋势、公司基本面、财务指标、估值、风险评估 |
| 行业研究 | 价值链分析、市场规模、竞争格局、政策环境、技术趋势、进入壁垒 |
| 投资尽职调查 | 商业模式、财务健康、管理评估、市场机会、风险因素、退出路径 |
| 竞争情报 | 竞争者识别、战略对比、SWOT分析、差异化定位、市场动态 |

### 步骤 1.2：选择分析框架与模型

根据识别的领域和研究主题，选择**一个或多个**专业分析框架来构建每章的推理。所选框架指导章节骨架中的**分析逻辑**（步骤 1.3）。

#### 战略与环境分析

| 框架 | 说明 | 最佳适用 |
|------|------|----------|
| **SWOT Analysis** | 优势、劣势、机会、威胁 | 品牌评估、竞争定位、战略规划 |
| **PEST / PESTEL Analysis** | 政治、经济、社会、技术（+环境、法律） | 宏观环境扫描、市场进入评估、政策影响分析 |
| **Porter's Five Forces** | 供应商议价能力、买方议价能力、新进入者威胁、替代品威胁、行业竞争 | 行业竞争格局、进入壁垒评估、利润率分析 |
| **Porter's Diamond Model** | 要素条件、需求条件、相关产业、企业战略与结构 | 国家/区域竞争优势分析 |
| **VRIO Analysis** | 价值、稀缺性、不可模仿性、组织性 | 核心竞争力评估、资源优势分析 |

#### 市场与增长分析

| 框架 | 说明 | 最佳适用 |
|------|------|----------|
| **STP Analysis** | 细分、目标、定位 | 市场细分、目标市场选择、品牌定位 |
| **BCG Matrix (Growth-Share Matrix)** | 明星、现金牛、问号、瘦狗 | 产品组合管理、资源配置决策 |
| **Ansoff Matrix** | 市场渗透、市场开发、产品开发、多元化 | 增长战略选择 |
| **Product Life Cycle (PLC)** | 导入期、成长期、成熟期、衰退期 | 产品策略制定、市场时机决策 |
| **TAM-SAM-SOM** | 总体/可服务/可获得市场 | 市场规模测算、机会量化 |
| **Technology Adoption Lifecycle** | 创新者 → 早期采用者 → 早期多数 → 晚期多数 → 落后者 | 新兴技术/品类渗透分析 |

#### 消费者与行为分析

| 框架 | 说明 | 最佳适用 |
|------|------|----------|
| **Consumer Decision Journey** | 认知 → 考虑 → 评估 → 购买 → 忠诚 | 消费者行为路径映射、触点优化 |
| **AARRR Funnel (Pirate Metrics)** | 获客、激活、留存、收入、推荐 | 用户增长分析、转化率优化 |
| **RFM Model** | 最近购买时间、购买频率、消费金额 | 客户价值细分、精准营销 |
| **Maslow's Hierarchy of Needs** | 生理 → 安全 → 社交 → 尊重 → 自我实现 | 消费者心理分析、产品价值主张 |
| **Jobs-to-be-Done (JTBD)** | 用户在特定场景中需要完成的"任务" | 需求洞察、产品创新方向 |

#### 财务与估值分析

| 框架 | 说明 | 最佳适用 |
|------|------|----------|
| **DuPont Analysis** | ROE = 净利润率 × 资产周转率 × 权益乘数 | 盈利能力分解、财务健康诊断 |
| **DCF (Discounted Cash Flow)** | 自由现金流折现 | 企业/项目估值 |
| **Comparable Company Analysis** | PE、PB、PS、EV/EBITDA 倍数比较 | 相对估值、同行基准对比 |
| **EVA (Economic Value Added)** | 税后营业利润 - 资本成本 | 价值创造能力评估 |

#### 竞争与战略定位

| 框架 | 说明 | 最佳适用 |
|------|------|----------|
| **Benchmarking** | 关键绩效指标逐项对比 | 竞争者差距分析、最佳实践识别 |
| **Strategic Group Mapping** | 沿两个关键维度聚类竞争者 | 竞争格局可视化、空白区域识别 |
| **Value Chain Analysis** | 基本活动 + 支持活动价值分解 | 成本优势来源、差异化机会识别 |
| **Blue Ocean Strategy** | 价值曲线、四步行动框架（消除-减少-提升-创造） | 差异化创新、新市场空间创造 |
| **Perceptual Mapping** | 沿两个消费者感知维度绘制品牌位置 | 品牌定位分析、市场空白发现 |

#### 行业与供应链分析

| 框架 | 说明 | 最佳适用 |
|------|------|----------|
| **Industry Value Chain** | 上游 → 中游 → 下游分解 | 行业结构理解、利润分布分析 |
| **Gartner Hype Cycle** | 技术触发 → 期望膨胀峰值 → 幻灭低谷 → 启蒙斜坡 → 生产力平台 | 新兴技术成熟度评估 |
| **GE-McKinsey Matrix** | 行业吸引力 × 竞争实力 | 业务组合优先级、投资决策 |

#### 选择原则

1. **领域优先**：基于步骤 1.1 识别的领域，从上述工具包中选择**2-4个**最相关的框架
2. **互补性**：选择互补而非重叠的框架（如宏观层用 PESTEL + 微观层用 Porter's Five Forces）
3. **深度优于广度**：深入应用2个框架胜过浅层堆叠6个
4. **数据可行性**：所选框架必须能被下游数据采集技能支持——如果框架所需数据无法合理获取，则降级或替换
5. **显式映射**：在章节骨架中，明确标注每章使用哪个框架及如何应用

#### 框架选择输出格式

```markdown
## 框架选择

| 章节 | 所选框架 | 应用方式 |
|------|----------|----------|
| 市场规模与增长趋势 | TAM-SAM-SOM + Product Life Cycle | TAM-SAM-SOM 量化市场空间，PLC 判断市场阶段 |
| 竞争格局评估 | Porter's Five Forces + Strategic Group Mapping | 五力模型评估行业竞争强度，群体映射可视化竞争定位 |
| 消费者画像 | RFM + Consumer Decision Journey | RFM 细分客户价值，决策旅程识别关键转化节点 |
| 品牌策略建议 | SWOT + Blue Ocean Strategy | SWOT 总结全局态势，蓝海战略引导差异化方向 |
```

### 步骤 1.3：设计章节骨架

产出层级化的章节结构。每章必须包含：

1. **章节标题** — 专业、简洁、基于主题（遵循格式部分的标题约束）
2. **分析目标** — 本章旨在揭示什么
3. **分析逻辑** — 推理链或框架（必须引用步骤 1.2 中选择的框架）
4. **核心假设** — 待数据验证或推翻的初步假设

#### 章节骨架输出格式

```markdown
## 分析框架

### 第1章：[标题]
- **分析目标**：[本章旨在...]
- **分析逻辑**：[使用的框架或推理链]
- **核心假设**：[待验证的假设]
- **数据需求**：（见步骤 1.4）
- **可视化计划**：（见步骤 1.5）

### 第2章：[标题]
...
```

### 步骤 1.4：定义各章数据查询需求

为每章指定**具体需要采集什么数据**。这是连接下游数据采集技能的桥梁。

每条数据需求必须包含：

| 字段 | 说明 |
|------|------|
| **数据指标** | 需要的具体指标或数据点（如"中国护肤市场规模 2020-2025（十亿元）"） |
| **数据类型** | 定量、定性或混合 |
| **建议来源** | 建议的来源类别：行业报告、财务报表、政府统计、社交媒体、电商平台、调查数据、新闻 |
| **搜索关键词** | 供数据采集代理使用的建议搜索查询 |
| **优先级** | P0（必需）/ P1（重要）/ P2（补充） |
| **时间范围** | 数据应覆盖的时间段 |

#### 数据需求输出格式（每章）

```markdown
#### 数据需求

| # | 数据指标 | 数据类型 | 建议来源 | 搜索关键词 | 优先级 | 时间范围 |
|---|----------|----------|----------|------------|--------|----------|
| 1 | 市场规模（十亿元） | 定量 | 行业报告、政府统计 | "中国护肤市场规模 2024" | P0 | 2020-2025 |
| 2 | 复合年增长率 | 定量 | 行业报告 | "护肤CAGR增长率" | P0 | 2020-2025 |
| 3 | 子品类份额 | 定量 | 电商平台、行业报告 | "护肤品类份额 面霜精华防晒" | P1 | 最新 |
| 4 | 政策与监管更新 | 定性 | 政府公告、新闻 | "化妆品法规 2024" | P2 | 近1年 |
```

### 步骤 1.5：定义各章可视化与内容结构

为每章指定**计划的可视化**和**内容结构**：

| 字段 | 说明 |
|------|------|
| **可视化类型** | 图表类型：折线图、柱状图、饼图、散点图、雷达图、热力图、桑基图、对比表等 |
| **可视化标题** | 图表的描述性标题 |
| **可视化数据映射** | 哪些数据指标映射到 X/Y 轴或分段 |
| **对比表设计** | 数据对比表的列标题和对比维度 |
| **论证结构** | 计划的"是什么 → 为什么 → 所以呢"叙事大纲 |

#### 可视化计划输出格式（每章）

```markdown
#### 可视化与内容计划

**图表1**：[类型] — [标题]
- X轴：[维度]，Y轴：[指标]
- 数据来源：对应数据需求 #1、#2

| 维度 | 项目A | 项目B | 项目C |
|------|-------|-------|-------|

**论证结构**：
1. **观察（是什么）**：[数据揭示的表面现象]
2. **归因（为什么）**：[驱动因素或深层原因]
3. **启示（所以呢）**：[战略启示或建议行动]
```

# 阶段一→二交接：数据采集与图表生成

分析框架生成后，交接给**其他数据采集技能**（如 deep-research、data-analysis、网络搜索代理）来：

1. 执行各章数据需求中的**搜索关键词**
2. 采集定量数据、定性洞察和来源 URL
3. 基于**可视化与内容计划**生成图表
4. 返回**数据包**，包含：
   - **数据摘要**：各章的原始数字、指标和定性发现
   - **图表文件**：生成的图表图片及本地文件路径
   - **外部搜索发现**：用于引用的来源 URL 和摘要

> **本技能不执行数据采集。** 仅产出框架（阶段一）和最终报告（阶段二）。
>
> **图表生成**：如果可视化/图表技能可用（如 data-analysis、image-generation），图表生成可推迟到阶段二开始时——见步骤 2.3。

# 阶段二：报告生成

## 目的

接收上游完成的**分析框架**和**数据包**，将其综合为最终的咨询级报告。

## 阶段二输入

| 输入 | 说明 | 必需 |
|------|------|------|
| **分析框架** | 阶段一产出的框架文档 | 是 |
| **数据摘要** | 按章组织的数据采集阶段收集的数据 | 是 |
| **图表文件** | 生成的图表图片的本地文件路径。如未提供，将在步骤 2.3 使用可用可视化技能生成 | 可选 |
| **外部搜索发现** | 用于行内引用的 URL 和摘要 | 可选 |

## 阶段二工作流程

### 步骤 2.1：接收并验证输入

验证所有必需输入是否存在：

1. **分析框架** — 确认包含章节骨架、数据需求和可视化计划
2. **数据摘要** — 确认包含按章组织的数据，与 P0 需求交叉验证
3. **图表文件** — 确认文件路径为有效的本地路径

如缺少任何 P0 数据，在报告中注明并向用户标记。

### 步骤 2.2：映射报告结构

从分析框架映射最终报告结构：

1. **摘要** — 包含关键要点的执行摘要
2. **引言** — 背景、目标、方法论
3. **正文章节（2...N）** — 从框架的章节骨架映射
4. **结论** — 纯客观综合
5. **参考文献** — GB/T 7714-2015 格式的参考文献

### 步骤 2.3：生成章节图表（报告前可视化）

在撰写报告之前，根据分析框架的**可视化与内容计划**生成所有计划图表。此步骤确保每个子章节在叙事写作开始前都有"视觉锚点"。

#### 何时执行此步骤

- **图表文件已提供**：跳过此步骤——直接进入步骤 2.4。
- **图表文件未提供但有可视化技能可用**：执行此步骤先生成所有图表。
- **无图表文件且无可视化技能可用**：跳过此步骤——在步骤 2.4 中使用对比表作为主要视觉锚点，并注明缺少图表。

#### 图表生成工作流程

1. **提取图表任务**：解析分析框架中所有 `可视化与内容计划` 条目，构建图表生成任务清单：

| # | 章节 | 图表类型 | 图表标题 | 数据映射 | 数据来源 |
|---|------|----------|----------|----------|----------|
| 1 | 2.1 | 折线图 | 2020-2025年市场规模趋势 | X: 年份, Y: 市场规模（十亿元） | 数据需求 #1、#2 |
| 2 | 3.1 | 饼图 | 消费者年龄分布 | 分段: 年龄组, 值: 占比% | 数据需求 #5 |
| ... | ... | ... | ... | ... | ... |

2. **准备图表数据**：对每个图表任务，从**数据摘要**中提取对应的数据点。
   > **关键**：仅使用数据摘要中提供的数字。不得为使图表更好看而编造或"平滑"数据。如数据点缺失，图表必须反映这一现实（如断线或缺失柱），或调整图表类型。

3. **委托可视化技能**：调用可用的可视化/图表技能（如 `data-analysis`）处理每个图表任务，提供：
   - 图表类型和标题
   - 结构化数据
   - 轴标签和格式偏好
   - 输出文件路径约定：`charts/chapter_{N}_{chart_index}.png`

4. **收集图表文件路径**：记录所有生成的图表文件路径，用于步骤 2.4 中的嵌入：

```markdown
## 已生成图表
| # | 章节 | 图表标题 | 文件路径 |
|---|------|----------|----------|
| 1 | 2.1 | 2020-2025年市场规模趋势 | charts/chapter_2_1.png |
| 2 | 3.1 | 消费者年龄分布 | charts/chapter_3_1.png |
```

5. **验证**：确认所有 P0 优先级图表已生成。如任何图表生成失败，注明并回退到该子章节的对比表。

> **原则**：在开始报告写作之前完成所有图表生成。这确保一致的视觉叙事，避免生成与写作交替进行。

### 步骤 2.4：撰写报告

对每个子章节，遵循**"视觉锚点 → 数据对比 → 综合分析"**流程：

1. **视觉证据块**：使用 `![图片描述](实际文件路径)` 嵌入图表——使用步骤 2.3 中收集的文件路径
2. **数据对比表**：创建关键指标的 Markdown 对比表
   > **来源规则**：表中的每个数字必须来自数据摘要。禁止虚构。
3. **综合叙事分析**：按"是什么 → 为什么 → 所以呢"撰写分析文本
   > **叙事规则**：叙事必须解释**提供的**数据。不得做出输入不支持的论断。

每个子章节必须以一段扎实的分析段落结尾（至少200字），该段落：
- 综合矛盾或相互印证的数据点
- 揭示底层用户张力或机会
- 可选地以一句精炼的"一句话真相"用引用块（`>`）收尾

### 步骤 2.5：最终结构自检

输出前，确认报告按顺序包含**所有章节**：

```
摘要 → 1. 引言 → 2...N. 正文章节 → N+1. 结论 → N+2. 参考文献
```

额外验证：
- 步骤 2.3 中生成的所有图表已嵌入正确的子章节
- `![](路径)` 引用中的图表文件路径有效
- 没有图表的子章节有对比表作为视觉锚点

报告**不得**在结论后停止——**必须**包含参考文献作为最后章节。

## 格式与语调标准

### 咨询语调
- **语调**：麦肯锡/BCG — 权威、客观、专业
- **语言**：所有标题和内容使用 `output_locale` 指定的语言
- **数字格式**：千位分隔符使用英文逗号（`1,000` 而非 `1，000`）
- **数据强调**：**加粗**重要观点和关键数字

### 标题约束
- **编号**：使用标准编号（`1.`、`1.1`）直接跟标题
- **禁止前缀**：不得使用"第X章"、"部分"、"节"作为前缀
- **允许的语调词**：分析、画像、概览、洞察、评估
- **禁止用词**："解码"、"DNA"、"秘密"、"心灵图景"、"太阳系"、"解锁"

### 子章节结论
- **要求**：每个子章节以一段扎实的分析段落结尾（至少200字）。
- **叙事流**：该段落必须看起来是正文的自然延续。必须将本节发现综合为战略判断。
- **内容逻辑**：
    1. 综合上述矛盾或相互印证的数据点。
    2. 揭示*底层*用户张力或机会。
    3. 关键洞察：**可选**：仅当你有简洁有力的"一句话真相"时，放在最末尾使用**引用块**（`>`）锚定本节。

### 洞察深度（"所以呢"链）

每条洞察必须连接**数据 → 用户心理 → 战略启示**：

```
❌ 差："女性占60%。策略：针对女性。"

✅ 好："女性占60%，TGI高达180。**这表明**
   购买决策由审美和社交验证驱动
   而非纯粹实用。**因此**，媒体投放应转向
   视觉密集型平台（如小红书/Instagram）以最大化点击率，
   将男性受众仅作为次要的送礼群体。"
```

### 参考文献
- **行内**：使用外部搜索发现时，用 markdown 链接标注来源（如 `[来源标题](URL)`）
- **参考文献部分**：严格按 **GB/T 7714-2015** 格式

### Markdown 规则
- **直接开始**：直接以 `# 报告标题` 开始——无引言文字
- **无分隔线**：不得使用水平线（`---`）

## 注意事项

- 本技能在多步骤代理工作流的**两个阶段**中运作：
  - **阶段一**产出分析框架和数据采集需求
  - **数据采集**由其他技能执行（deep-research、data-analysis 等）
  - **阶段二**接收采集的数据并产出最终报告
- 动态标题：将框架中的主题**改写**为专业、简洁的基于主题的标题
- 结论部分**不得**包含详细建议——建议应属于前面的正文章节
- **零虚构政策**：报告中的每条陈述、图表和数字必须有输入数据摘要中的数据点支持。如果数据缺失，坦诚说明。
- **可溯源性**：如被要求，必须能指出数据摘要或外部搜索发现中支持某论断的具体行。
- 框架应根据特定领域调整分析维度和深度（财务分析使用不同于消费者洞察的框架）
- 当研究主题模糊时，默认采用最广的合理范围并注明假设
